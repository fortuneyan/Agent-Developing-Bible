## 这本书是怎么来的

2023年底，我们团队开始做第一个真正意义上的 Agent——不是那种"LLM 包一层接口"的玩具，而是能自主规划、调用工具、持久记忆、多轮迭代的系统。

那段时间我们踩了很多坑：Context 窗口爆了不知道怎么压缩、工具调用返回格式对不上、长期记忆存进去但召回总是找不到最相关的、多个 Agent 协作时一个失败把整条链路都拖垮……

网上的教程要么太浅（"三行代码搞定 Agent"）、要么太散（把 LangChain 文档翻译一遍）。我们想要的是**一本能让人少踩坑、快速到达"真正可用"级别的实战手册**。

这本书就是从那个需求里长出来的。

## 一个关键的认知对齐

很多人初次接触 Agent 会有一个误区：**觉得 Agent 就是"更聪明的 Chatbot"**——给它更多工具、更长的 Prompt，它就更厉害。

实际上不是这样的。

Chatbot 是**反应式**的：你问，它答，完事儿。

Agent 是**目标驱动式**的：你给它一个目标，它自己规划步骤、调用工具、处理中间结果、处理失败、再规划，直到达成目标或判断无法完成。

用一句更简练的工程语言来说：**智能体在循环中运行工具以实现目标**——循环、工具、目标，缺一不可。

这个区别导致了完全不同的工程复杂度：

| 维度 | Chatbot | Agent |
| :--- | :--- | :--- |
| 状态管理 | 基本无 / 简单对话历史 | 复杂，跨多轮、跨工具 |
| 错误处理 | LLM 自己想办法 | 需要显式的重试、降级、回滚 |
| 可观测性 | 看 prompt + response 就够 | 需要追踪整条执行链路 |
| 成本控制 | 按对话轮次估算 | 需要细粒度到每个子任务 |
| 测试难度 | 相对可测 | 非确定性强，测试体系要专门设计 |

这不是在劝退你，而是告诉你：**做好 Agent 需要的工程思维，跟做好一个后端服务差不多复杂**，只是维度不一样。本书会把这些维度一一讲清楚。

## 本书的结构

全书分为四个模块，十四章，另有附录：

### 模块一：基础设施（第 1-4 章）

这四章解决"让 Agent 能跑起来"的基础问题。

- **第1章 AI 平台与 API Key 管理**：多供应商接入、密钥安全、费用追踪，这些不起眼的事情在生产环境里往往是第一个出问题的地方。

- **第2章 Context 与 Prompt 管理**：Context 是 Agent 的"工作记忆"，Prompt 是它的"行为指令"。这两个管好了，Agent 的表现能提升一个档次。

- **第3章 模型数据与格式化管理**：LLM 的输出本质上是字符串，怎么让它可靠地输出结构化数据是个系统性工程问题。

- **第4章 Function Calling**：工具调用是 Agent 从"说"到"做"的桥梁，这里讲清楚它的完整机制和工程陷阱。

### 模块二：能力扩展（第 5-9 章）

这五章解决"让 Agent 更强"的问题。

- **第5章 Agent Skills**：如何组织、复用、动态加载 Agent 的技能体系。

- **第6章 短期记忆（Session Memory）**：对话级别的记忆，看起来简单，但压缩策略和检索效率里有不少细节。

- **第7章 长期记忆**：跨会话持久化记忆，向量数据库选型、数据生命周期、用户隔离，这些都要认真对待。

- **第8章 RAG 检索增强生成**：知识库问答的核心技术，重点讲评估体系和性能优化，避开"RAG 上线后效果一言难尽"的坑。

- **第9章 规划与推理**：从 ReAct 到 Plan-and-Execute，以及如何防止 Agent 陷入规划死循环。

### 模块三：协作与落地（第 10-13 章）

这四章解决"让 Agent 在生产环境活下去"的问题。

- **第10章 多智能体治理**：多个 Agent 协作时的监控、测试与运维，这是本书技术难度最高的一章。

- **第11章 Pipeline 流程编排**：复杂业务流程的编排模式，包括串行、并行、条件分支、循环。

- **第12章 主流框架与生态**：LangChain、LangGraph、LlamaIndex、AutoGen 等框架的选型指南，以及升级策略。

- **第13章 平台纵览**：从 GPTs 到 Dify，开发平台的选型、迁移和用户体验设计。

### 模块四：自我进化（第 14 章）

这一章探索 Agent 从"工具"进化为"伙伴"的前沿实践。

- **第14章 自我进化**：基于记忆的经验学习、基于基因协议的知识共享、基于沙箱的代码自我修改，以及完整的自我进化工程实践。

### 附录

- **附录A 术语解释**：全书核心术语的完整定义、首次定义章节、典型使用场景和快速查找表，方便读者查阅。

## 怎么读这本书

**如果你是从零开始做 Agent**：建议顺序读，第 1 章是基础，不要跳过。

**如果你已经有一个在跑的 Agent，想优化它**：可以直接跳到痛点章节：

- 检索效果差 → 第 8 章

- Agent 经常跑歪 → 第 9 章（规划）、第 10 章（监控）

- 多 Agent 协作乱 → 第 10 章

- 框架选不好 → 第 12 章

- 成本太高 → 第 2 章（Context 压缩）、第 10 章（成本控制）

- 想让 Agent 自主学习和进化 → 第 14 章

**关于代码示例**：

书中的代码都是可以直接运行的 Python，不是伪代码。依赖库尽量用标准库或业界主流的稳定版本。每段代码后面会说明实际踩坑经验，这些往往比代码本身更有价值。
